# SUSHI UA — сайт-меню (стиль ChoiceQR)

Статичний сайт-каталог меню для закладу: обкладинка, категорії, перегляд
страв, кошик і замовлення в **Instagram Direct** / Telegram. Без бекенду
й абонплати — працює як звичайний статичний сайт (GitHub Pages, Vercel
тощо).

## Файли

- **`index.html`** — структура сторінки
- **`styles.css`** — стилі (темна тема: чорний + матове золото, адаптив)
- **`script.js`** — меню, навігація, кошик, замовлення, контакти

## Можливості

- 🏷️ **Обкладинка + інфо закладу** (назва, адреса, спосіб отримання)
- 🍽️ **Способи отримання** — клікабельні плитки «В закладі / З собою /
  Доставка» (підсвічуються, потрапляють у замовлення)
- 🗂️ **Меню з категоріями (drill-down)**: спершу список категорій, потім —
  страви обраної категорії з підкатегоріями та липкою навігацією (scroll-spy)
- ⭐ **Популярне** та ❤ **Обране** (сердечка зберігаються в браузері)
- 🔍 **Пошук** по меню в шапці
- 🪟 **Картка страви**: добавки (авокадо, ікра, соуси…) + «схожі страви»
- 🛒 **Кошик** із сумою, кількістю, добавками
- 📲 **Замовлення в Instagram Direct** (копіює текст + відкриває чат),
  Telegram (за прапорцем) і «Скопіювати замовлення»
- ☰ **Бічна панель**: контакти, соцмережі, «корисна інформація»
- 📷 **Галерея** + 🗺️ **Google-карта** + графік роботи з розкладом
- 🖼️ **Фото страв** з фолбеком на іконку (без «битих» картинок)
- 📱 Повна адаптація під телефон

## Як налаштувати під себе (усе в `script.js`)

### 1. Контакти та локація — блок `CONFIG` на початку файлу

```js
const CONFIG = {
  shopName: "SUSHI UA",
  instagram: "sushiua_",                  // нік в Instagram (без @)
  telegram: "sushiua_",                   // нік у Telegram (без @)
  telegramEnabled: false,                 // true — увімкнути кнопку Telegram

  phone: "+380XXXXXXXXX",                 // телефон для посилання
  phoneLabel: "+38 (0XX) XXX-XX-XX",      // як показувати телефон
  address: "вул. ..., місто",             // адреса закладу
  mapQuery: "точна адреса або координати", // для Google-карти
  hours: { dinein:"10:00–22:00", takeaway:"10:00–22:00", delivery:"10:00–22:00" },
};
```

### 2. Категорії та групи

```js
// Підкатегорії (внутрішня назва -> підпис)
const CATEGORIES = [
  { key:"pizza", name:"Піца" },
  // ...
];

// Групи верхнього рівня (що видно у списку категорій)
const GROUPS = [
  { key:"pizza", name:"Піца", subs:["pizza"] },         // одна підкатегорія
  { key:"drinks", name:"Напої", subs:["cold","hot"] },  // кілька підкатегорій
  // ...
];
```

### 3. Страви — масив `MENU`

```js
{ cat:"pizza", title:"Маргарита", desc:"Томат, моцарела, базилік · 30 см", price:180, badge:"Хіт" },
```

- `cat` — `key` підкатегорії з `CATEGORIES`
- `desc` — склад; вагу/розмір пишіть через `·` (напр. `· 300 г`) —
  вона автоматично виноситься окремим рядком
- `badge` — необов'язкова позначка («Хіт», «Новинка»)
- `img` — необов'язкове посилання на власне фото страви

> Демо-позиції позначені `badge:"Демо"` — їх треба замінити на реальні.

### 4. Фото

Фото за замовчуванням задаються по категорії в `CATEGORY_IMG` (Unsplash).
Щоб поставити власне фото конкретній страві — додайте їй поле `img:"...URL..."`.
Якщо фото не завантажиться, автоматично покажеться елегантна іконка.

### 5. Добавки — масив `ADDONS`

```js
{ id:"av", title:"Авокадо", price:15 },
```

## Як подивитись

Відкрий `index.html` у браузері — нічого встановлювати не треба.
Карта, шрифти та фото підвантажуються з інтернету.

## Що замінити на справжнє

- Меню: реальні назви, ціни, склад, вага (масив `MENU`)
- Фото страв (поле `img` або `CATEGORY_IMG`)
- Контакти, адреса, карта, графік (блок `CONFIG`)
- Обкладинку (`<img class="cover__img">` в `index.html`)

> Демонстраційний макет. Онлайн-оплата та адмінка замовлень потребують
> окремого бекенду — тут замовлення приймаються в Instagram Direct/Telegram.
